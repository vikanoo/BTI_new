## Вторник 31.03 — PDF-отчёт пользователю

**Цель дня:** Пользователь получает профессиональный PDF-документ с результатами анализа в дополнение к текстовому сообщению.

### Python-сервис: новый эндпоинт `POST /generate-report`

- [ ] Добавить эндпоинт в `python-service/app.py`:
  - Вход: `multipart/form-data` — `image` (PNG аннотированного плана) + `report_json` (JSON с данными отчёта)
  - Логика: сформировать PDF через библиотеку `reportlab`:
    - Заголовок: "Результат анализа перепланировки"
    - Дата анализа
    - Аннотированный план БТИ (вставить PNG)
    - Таблица изменений: № / Помещение / Тип изменения / Классификация / Рекомендация
    - Итоговое резюме
    - Дисклеймер
  - Выход: PDF-файл (бинарный ответ, mimetype `application/pdf`)
- [ ] Добавить `reportlab` в `python-service/requirements.txt`
- [ ] Обновить `python-service/Dockerfile` (зависимости подтянутся автоматически)

### Интеграция в N8N

- [ ] После `HTTP Request - Annotate Changes` добавить ветку генерации PDF:
  - `HTTP Request - Generate Report` — POST к `http://python-service:5000/generate-report` с аннотированным планом и данными отчёта
  - `Telegram - Send PDF Report` — отправить PDF через `sendDocument` (подпись: "📄 PDF-отчёт об анализе перепланировки")
- [ ] Порядок сообщений пользователю:
  1. Аннотированный план с изменениями (фото)
  2. PDF-отчёт (документ)
  3. Текстовое резюме

- [ ] Пересобрать контейнер на VPS: `docker-compose up -d --build python-service`

**Результат дня:** Пользователь получает полноценный PDF-документ с планом и таблицей нарушений
